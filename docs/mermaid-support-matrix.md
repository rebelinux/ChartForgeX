# Mermaid Support Matrix

This matrix is the working contract for ChartForgeX Mermaid support. A family is complete only when it has source-preserving parsing, typed semantic models where rendering is implemented, diagnostics for unsupported syntax, deterministic SVG and PNG output where renderable, Markdown fence coverage, Mermaid.js conformance fixtures, and smoke tests that prove the public artifact contract.

The current compatibility reference is the Mermaid.js version pinned by `tests/mermaid-conformance/package.json`.

## Recognized But Not Renderable Yet

These families are detected and returned as `MermaidDocument` with raw body statements in `RawStatements`, plus a warning diagnostic. They should not emit a visual artifact until a semantic model and deliberate ChartForgeX mapping exist.

| Family | Header | Current state | Best next target | Notes |
| --- | --- | --- | --- | --- |
| ZenUML | `zenuml` | Recognized, raw statements retained, warning diagnostic | Sequence-like artifact after grammar evaluation | Keep separate from Mermaid sequence until semantic differences are understood. |

## Priority Order

1. Complete conformance fixtures and smoke coverage for every implemented family before claiming stronger Mermaid compatibility.
2. Keep current non-renderable Mermaid families recognized as diagnostic-only until a semantic model exists.
3. Harden Ishikawa/fishbone and Wardley rendering with Mermaid layout/style parity and richer examples while keeping `FishboneDiagramBlock` and `WardleyMapBlock` product-neutral.
4. Harden C4, git graph, block, Venn, and architecture rendering with richer docs examples now that typed models exist.
5. Leave ZenUML as diagnostic-only until there is a reusable product-neutral model for it.
