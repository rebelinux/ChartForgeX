# Topology Icon Packs

Topology icon packs are portable stencil libraries for ChartForgeX topology diagrams. A pack can be handwritten, generated by a product adapter, or imported from a vendor/community SVG source. The recommended generated layout keeps manifests readable and assets editable:

```text
my-vendor-pack/
  LICENSE
  SOURCE.md
  _reports/
    import-report.json
  networking/
    manifest.json
    svg/
      firewall.svg
      switch.svg
    previews/
      firewall.png
      switch.png
```

`manifest.json` is the contract ChartForgeX loads. `svg/*.svg` is the source artwork the renderer uses after the manifest is loaded. `previews/*.png` is optional picker/gallery artwork and should be treated as generated output. A host can ship only a manifest with inline SVG for small custom packs, but sidecar SVG files are preferred for large vendor packs because reviews stay readable and designers can edit artwork without touching JSON.

## Manifest Artwork

Use `artwork.svgPath` for pack-local SVG sidecars and `artwork.previewPath` for optional thumbnails:

```json
{
  "schema": "chartforgex.topology.iconPack",
  "version": 1,
  "id": "acme-network",
  "label": "Acme Network",
  "vendor": "Acme",
  "metadata": {
    "source.url": "https://example.com/acme-icons",
    "source.revision": "2026.05.09",
    "source.license": "MIT"
  },
  "icons": [
    {
      "id": "firewall",
      "label": "Acme Firewall",
      "nodeKind": "Gateway",
      "shape": "Firewall",
      "symbol": "FW",
      "color": "#D92D20",
      "category": "Network",
      "artwork": {
        "svgViewBox": "0 0 48 48",
        "svgPath": "svg/firewall.svg",
        "previewPath": "previews/firewall.png"
      },
      "tags": [ "firewall", "security", "gateway" ]
    }
  ]
}
```

Asset paths are intentionally relative to the manifest directory. Do not use absolute paths, parent-directory traversal, file URLs, or remote URLs for sidecar artwork. ChartForgeX validates paths before resolving them and loads sidecar SVGs through the same SVG sanitizer used by the importer.

## Import Tool

`ChartForgeX.Tools.IconImport` converts a folder tree of SVG files into ChartForgeX-compatible packs. It is tooling only; SkiaSharp and Svg.Skia are not runtime dependencies of `ChartForgeX`.

```powershell
dotnet run --project .\ChartForgeX.Tools.IconImport\ChartForgeX.Tools.IconImport.csproj -c Release -- `
  --source C:\Source\VendorStencils `
  --output .\assets\topology-icons\vendor-stencils `
  --source-url https://example.com/vendor/stencils `
  --source-revision 2026.05.09 `
  --source-license MIT `
  --source-license-url https://example.com/vendor/stencils/LICENSE `
  --source-license-path LICENSE `
  --vendor Vendor `
  --pack-id-prefix vendor-stencils `
  --pack-label-prefix "Vendor Stencils" `
  --generate-png `
  --preview-size 128
```

The importer:

- creates one generated pack per top-level source category that contains SVG files
- writes sanitized `svg/*.svg` sidecars instead of embedding large SVG strings in JSON
- optionally writes `previews/*.png` thumbnails for stencil browsers and picker UIs
- records pack-level provenance such as source URL, source revision, license, and license URL
- records icon-level provenance such as source path, source file name, and detected viewBox
- skips unsafe SVGs rather than importing scriptable or externally dependent artwork
- writes `SOURCE.md` and `_reports/import-report.json` so an asset PR can be audited later without placing non-manifest JSON beside pack manifests

Use `--help` to print the current command-line options.

## Review Checklist

For a generated vendor pack PR, verify:

- the upstream source revision is pinned
- the source license is copied or clearly linked
- generated manifests do not contain local machine paths
- generated manifests use `artwork.svgPath` instead of large inline `artwork.svg` payloads
- `svg/*.svg` files are sanitized, pack-local, and reviewable
- `previews/*.png` files are generated artifacts, not runtime requirements
- ChartForgeX core builds without adding asset-conversion dependencies

Useful checks:

```powershell
git diff --check
rg -n '"svg"\s*:' assets\topology-icons\vendor-stencils
rg -n "C:\\|AppData|Temp" assets\topology-icons\vendor-stencils
dotnet build .\ChartForgeX\ChartForgeX.csproj -c Release --no-restore -p:UseSharedCompilation=false -m:1 -nr:false
dotnet build .\ChartForgeX.Tools.IconImport\ChartForgeX.Tools.IconImport.csproj -c Release --no-restore -p:UseSharedCompilation=false -m:1 -nr:false
```
